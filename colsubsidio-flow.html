Hackathon Colsubsidio 2026

Perfilamiento Inteligente de Leads

Que los leads pagos se parezcan a los orgánicos: un pipeline de IA que aprende de los compradores reales para atraer a los correctos.

⚠

El problema

Colsubsidio invierte en pauta digital para múltiples proyectos de vivienda, pero solo una fracción de leads pagos tiene capacidad real de compra. Además, cada proyecto encaja con un segmento distinto y hoy no hay un sistema que dirija al lead correcto al proyecto correcto. Los orgánicos convierten bien porque llegan motivados al proyecto que les sirve. El reto: replicar eso con pauta.

Fase 1 — Investigación
01
Recolección de datos de clientes orgánicos
Base de compradores reales de vivienda Colsubsidio
+

Qué se hace

Consolidar todos los datos de clientes que compraron vivienda de forma orgánica: datos demográficos, historial de interacción, canal de llegada, tipo de inmueble comprado, subsidios aplicados, y comportamiento previo a la compra.

Requerimientos técnicos

  • Acceso al CRM de Colsubsidio (Salesforce / HubSpot / interno)
  • Exportar datos del ERP con histórico de ventas inmobiliarias
  • Acceso a Google Analytics / Meta Ads para tracking de touchpoints
  • Pipeline de ingesta: Python + Pandas para limpiar y unificar fuentes
  • Base de datos unificada: PostgreSQL o BigQuery

Rol de la IA

Claude procesa datos no estructurados (notas de vendedores, correos, llamadas) y los convierte en atributos estructurados del cliente usando NLP.

Entrada / Salida

Input

CRM, ERP, Analytics, registros de ventas, notas de asesores

Output

Dataset limpio de clientes orgánicos con +30 atributos por cliente

02
Análisis cuantitativo y cualitativo
Entender qué motiva la compra y qué patrones se repiten
+

Qué se hace

  • Cuantitativo: Rango salarial, estrato, edad, composición familiar, tipo de subsidio, ticket promedio, tiempo desde primer contacto hasta compra
  • Cualitativo: Motivaciones de compra (independencia, inversión, familia, cambio de ciudad), objeciones frecuentes, lenguaje usado en conversaciones, canales preferidos
  • Cruce: qué combinaciones de atributos tienen mayor tasa de conversión

Requerimientos técnicos

  • Análisis estadístico: Python (scikit-learn, scipy)
  • Clustering: K-Means o DBSCAN para agrupar perfiles similares
  • Análisis de sentimiento en notas/llamadas con Claude API
  • Visualización exploratoria: Plotly / Streamlit
  • Feature importance para identificar variables predictivas de conversión

Rol de la IA

Claude analiza transcripciones de llamadas y notas de venta para extraer motivaciones y objeciones. Genera resúmenes temáticos y encuentra patrones no obvios entre los perfiles que compran.

Entrada / Salida

Input

Dataset limpio, notas de asesores, transcripciones, encuestas post-venta

Output

Clusters de clientes, ranking de variables predictivas, mapa de motivaciones

Fase 2 — Motor de IA
03
Generación de Buyer Personas con IA
Perfiles dinámicos que se actualizan con cada venta
+

Qué se hace

La IA toma los clusters del paso anterior y construye buyer personas detallados: no solo demográficos, sino con motivaciones, miedos, objeciones, canales preferidos, horarios de actividad digital, tipo de contenido que consume, y las combinaciones de atributos que mejor predicen conversión.

Requerimientos técnicos

  • Claude API con prompt estructurado que recibe clusters y genera personas
  • Template de buyer persona con campos obligatorios: nombre ficticio, demografía, motivación primaria, objeciones, canal preferido, tipo de contenido, rango de capacidad adquisitiva, timeline de compra
  • Sistema de versionado: las personas se actualizan con cada venta nueva
  • Base de datos de personas: JSON/NoSQL para consulta rápida

Rol de la IA

Claude es el motor central: recibe datos agregados de los clusters, genera narrativas de buyer persona, y propone el mix óptimo de características para cada segmento. Se re-entrena con cada ciclo de ventas.

Entrada / Salida

Input

Clusters, variables predictivas, mapa de motivaciones, datos de conversión

Output

3-6 Buyer Personas con mix de atributos, scoring de relevancia por proyecto

04
Mapeo de proyectos × segmentos
Cada proyecto de vivienda se cruza con las personas que mejor le encajan
+

Qué se hace

  • Catalogar todos los proyectos activos de vivienda Colsubsidio con sus atributos: ubicación, estrato, rango de precio, tipo de unidad (apto, casa), metraje, subsidios aplicables, amenities, fecha de entrega
  • Cruzar cada proyecto contra los buyer personas: ¿qué persona encaja naturalmente con cada proyecto?
  • Generar una matriz proyecto × persona con score de afinidad: no todos los proyectos se muestran a todos, cada persona ve el que más le conviene
  • Definir reglas de negocio: inventario disponible, prioridad comercial, margen por proyecto

Requerimientos técnicos

  • Catálogo de proyectos estructurado: API del ERP inmobiliario o base propia
  • Cada proyecto tiene un vector de atributos comparable con el vector de cada persona
  • Algoritmo de afinidad: cosine similarity o weighted scoring entre proyecto y persona
  • Dashboard de afinidad: qué proyectos le pegan a qué personas y con qué score
  • Sincronización con inventario en tiempo real: si se agota un tipo de unidad, se reasigna

Rol de la IA

Claude analiza los atributos de cada proyecto y los cruza con las motivaciones y capacidad de cada buyer persona. Genera la narrativa de por qué un proyecto específico es ideal para una persona específica, y esa narrativa alimenta el contenido de la fase siguiente.

Entrada / Salida

Input

Catálogo de proyectos con atributos, buyer personas con scoring, inventario disponible

Output

Matriz proyecto × persona con score de afinidad + narrativa de match por combinación

05
Banco de características para contenido
Matriz de atributos que la IA combina para cada pieza
+

Qué se hace

  • Crear un banco de componentes modulares: hooks de texto, visuales de proyecto, beneficios clave, CTAs, tonos de comunicación, formatos (carrusel, video corto, render, testimonio)
  • Cada componente está tagueado por buyer persona, etapa del funnel, y canal
  • La IA puede combinar estos componentes en miles de variantes únicas

Requerimientos técnicos

  • Base de datos de assets tagueados: Airtable / Supabase
  • Sistema de templates para cada formato: email, ad imagen, ad video, landing
  • API de generación de imágenes: DALL·E / Midjourney API / Stable Diffusion
  • API de generación de video: Runway / Pika / HeyGen
  • Motor de combinación: lógica que recibe persona + proyecto + canal → genera pieza

Rol de la IA

Claude decide la combinación óptima de componentes para cada persona y canal. Genera copy, sugiere ángulos visuales, adapta el tono. Funciona como un director creativo automatizado.

Entrada / Salida

Input

Buyer personas, catálogo de proyectos, banco de assets, reglas de marca

Output

Matriz de combinaciones priorizadas por persona × canal × etapa

Fase 3 — Activación
06
Generación de contenido hiperpersonalizado
Videos, imágenes, textos y emails por buyer persona
+

Qué se hace

  • Imágenes: Renders del proyecto adaptados al estilo de vida de cada persona (familia, pareja joven, inversionista)
  • Videos: Clips cortos con narrativa personalizada por motivación (seguridad, cercanía al trabajo, valorización)
  • Textos: Copy de ads con hooks específicos por objeción y motivación
  • Emails: Secuencias de nurturing adaptadas al timeline de compra del lead

Requerimientos técnicos

  • Claude API para generación de copy y estructura de emails
  • DALL·E / Stable Diffusion para variaciones de imágenes
  • HeyGen / Synthesia para videos con avatar personalizado
  • Plataforma de email marketing: SendGrid / Mailchimp API
  • Sistema de versionado de creativos: cada pieza tiene ID + tags de persona/canal
  • Generación batch: producir 50-200 variantes por campaña

Rol de la IA

Claude orquesta todo el flujo: escribe los briefs creativos, genera el copy, define los prompts para imágenes/video, y valida que cada pieza sea coherente con la persona y el proyecto inmobiliario.

Entrada / Salida

Input

Matriz de combinaciones, assets del proyecto, guía de marca, personas

Output

Pack de creativos: 50-200 variantes tagueadas por persona × canal × formato

07
Pauta digital con testing multivariant
Desplegar combinaciones y medir cuáles convierten
+

Qué se hace

  • Lanzar las variantes de creativos como campañas en Meta, Google, TikTok con audiences lookalike basados en los clusters orgánicos
  • Testing A/B/n: cada combinación (imagen + copy + CTA + formato) compite contra las demás
  • Medir: CTR, costo por lead, tasa de calificación, engagement por pieza
  • Reasignar presupuesto automáticamente hacia las combinaciones ganadoras

Requerimientos técnicos

  • Meta Ads API + Google Ads API para despliegue programático
  • UTM tracking con parámetros de persona + variante + canal
  • Dashboard de performance en tiempo real: Looker Studio / Metabase
  • Reglas de auto-optimización: pausar variantes con CPA > umbral
  • Pixel + Conversion API para tracking server-side

Rol de la IA

Claude analiza el rendimiento en tiempo real, identifica qué combinaciones están ganando y por qué, y sugiere nuevas variantes basadas en los patrones exitosos. Aprende qué resuena con cada persona.

Entrada / Salida

Input

Pack de creativos, presupuesto, audiences lookalike, reglas de negocio

Output

Ranking de combinaciones ganadoras, leads entrantes con metadata de variante

Fase 4 — Conversión
08
Scoring inteligente de leads
Capacidad adquisitiva, intención, timeline y match
+

Qué se hace

  • Capacidad adquisitiva: Estimar ingresos por datos de formulario + enriquecimiento con datos abiertos + subsidio aplicable
  • Intención: Scoring basado en interacciones (qué piezas vio, cuántas veces, tiempo en landing, si descargó brochure)
  • Timeline: Clasificar si busca comprar en 0-3, 3-6, 6-12, o +12 meses
  • Match con proyecto: Qué tan bien encaja el lead con los proyectos disponibles
  • Score compuesto: leads se clasifican como Hot / Warm / Cold

Requerimientos técnicos

  • Modelo de scoring: XGBoost / LightGBM entrenado con datos de orgánicos
  • Features: interacciones + formulario + enriquecimiento + comportamiento web
  • Integración con CRM para push de leads calificados al equipo comercial
  • Formulario inteligente progresivo: solo pide lo que falta
  • Enriquecimiento de datos: API de cédula / RUES para validación

Rol de la IA

Claude enriquece el perfil del lead con análisis de sus interacciones: qué piezas lo atrajeron revela sus motivaciones. Predice el timeline de compra y sugiere qué proyecto mostrarle primero.

Entrada / Salida

Input

Lead con datos de formulario + historial de interacciones + metadata de variante

Output

Lead score (Hot/Warm/Cold) + proyecto recomendado + brief para el asesor

09
Remarketing inteligente para perfiles validados
Re-impactar a los leads con capacidad real hasta que estén listos para comprar
+

Qué se hace

  • Todos los leads que pasaron el scoring como aptos para compra (capacidad adquisitiva + interés validado) entran a un pool de remarketing segmentado
  • Hot leads: Secuencia urgente — recorrido virtual del apto, simulación financiera personalizada, agenda con asesor. Frecuencia alta (24-72h)
  • Warm leads: Nurturing por timeline — contenido de valorización, testimoniales de compradores similares, alertas de inventario ("quedan 3 unidades de tu tipo"). Frecuencia media (semanal)
  • Cold con capacidad: Remarketing de largo plazo — contenido educativo sobre compra de vivienda, cambios en tasas de interés, nuevos lanzamientos de proyectos que encajen. Frecuencia baja (quincenal)
  • Cada segmento recibe piezas distintas adaptadas a su proyecto ideal (de la matriz proyecto × persona)
  • Canales: Meta remarketing, Google Display, email drip, WhatsApp secuenciado, YouTube pre-roll

Requerimientos técnicos

  • Audiences personalizadas en Meta Ads y Google Ads sincronizadas con el CRM por score
  • Customer Match: subir listas de leads validados para remarketing cross-platform
  • Secuencias de email automatizadas: SendGrid / ActiveCampaign con triggers por comportamiento
  • WhatsApp Business API con templates aprobados y secuencia por timeline del lead
  • Frequency capping inteligente: no sobre-impactar, ajustar frecuencia según engagement
  • Pixel de retargeting + Conversion API para tracking sin cookies
  • Reglas de supresión: sacar del pool cuando agenda cita, compra, o pide no ser contactado

Rol de la IA

Claude decide qué pieza mostrar a cada lead en cada momento basándose en su historial de interacciones, qué contenido ya vio, qué lo enganchó más, y cuánto tiempo lleva en el funnel. Adapta el mensaje según la etapa: un lead warm que lleva 3 semanas recibe un ángulo distinto al de la semana 1. También detecta señales de compra (visitas repetidas a simulador, apertura de emails de precio) y escala leads de warm a hot automáticamente.

Entrada / Salida

Input

Leads validados con score + historial de interacciones + matriz proyecto × persona + inventario

Output

Secuencias activas por canal, escalamiento automático de leads, señales de compra al equipo comercial

10
Match final: apartamento → persona correcta
El inmueble ideal llega a quien puede y quiere comprarlo
+

Qué se hace

  • Cruzar el perfil enriquecido del lead (capacidad + motivación + timeline) con el inventario de unidades disponibles
  • Generar una recomendación personalizada: "Este es tu apartamento" con render personalizado, plan de pagos simulado, y beneficios relevantes
  • Email/WhatsApp automatizado con la recomendación antes de que el asesor llame
  • Brief para el asesor comercial: qué sabe el lead, qué lo motiva, qué objeciones anticipar

Requerimientos técnicos

  • Motor de recomendación: filtrado colaborativo + reglas de negocio
  • Integración con inventario de unidades: API del ERP inmobiliario
  • Generación de simulación financiera: cuota, subsidio, separación
  • Landing personalizada por lead: Next.js + dynamic routes
  • Integración WhatsApp Business API para envío automatizado
  • Dashboard de asesor con ficha del lead pre-armada

Rol de la IA

Claude genera la narrativa personalizada de venta: escribe el email/WhatsApp con el tono correcto, arma el brief del asesor, y genera el prompt para el render personalizado del apartamento en el contexto de vida del lead.

Entrada / Salida

Input

Lead calificado + inventario + scoring + buyer persona match

Output

Recomendación personalizada enviada al lead + ficha para el asesor comercial

Resultado esperado

Los leads pagos llegan al equipo comercial con el mismo nivel de perfilamiento y calidad que los orgánicos. Menos desperdicio de pauta, más conversiones reales.
